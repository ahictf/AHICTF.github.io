Cross-site Request Forgery (CSRF)

ก่อนอื่นเรามาทำความรู้จักกับช่องโหว่ CSRF หรือ Cross-site Request Forgery ซึ่ง CSRF เป็นช่องโหว่ที่เกิดจากผู้ไม่หวังดีสามารถบังคับให้ Browser ของผู้ใช้งานส่ง HTTP request ให้กับ Web Application กระทำบางอย่างที่อาจจะเป็นอันตรายต่อผู้ใช้งาน หลักการของ CSRF เป็นตัวอย่างดังรูป

ตัวอย่างการโจมตี Web Application ที่ไม่ได้ป้องกัน CSRF

หน้าเว็บที่เรานำมาทดสอบโจมตีช่องโหว่ CSRF เป็นหน้าเว็บที่เกี่ยวกับการเปลี่ยนรหัสผ่านของผู้ดูแลระบบ

ฝั่งผู้ดูแลระบบ

เมื่อผู้ดูแลระบบดำเนินการใส่รหัสผ่าน และกดปุ่ม Change เพื่อดำเนินการเปลี่ยนแปลงรหัสผ่าน จะตรวจพบว่ามีการส่งรหัสผ่านที่ผู้ดูแลระบบต้องการจะเปลี่ยนไปทำงาน

ผู้ดูแลระบบสามารถเปลี่ยนแปลงรหัสผ่านได้สำเร็จ โดยรหัสผ่านคือ ahictf1234

ฝั่งผู้ไม่หวังดี

หากผู้ไม่หวังดีทราบถึง HTTP request ส่วนของการเปลี่ยนรหัสผ่าน ผู้ไม่หวังดีดำเนินการสร้างเว็บไซต์ที่เอาไว้หลอกผู้ดูแลระบบให้เข้าใช้งาน

จากโค้ด HTML ดังกล่าวจะมีการแสดงข้อความ Hello, Free Money และมีการฝังคำสั่งเรียกใช้งานรูปภาพโดยฝัง URL ให้ไปเรียกใช้งานในส่วนของการเปลี่ยนรหัสผ่านของผู้ดูแลระบบ โดยเปลี่ยนรหัสผ่านจาก ahictf1234 เป็น code4sec

ผู้ไม่หวังดีดำเนินการทดสอบเข้าเว็บไซต์ดังกล่าวก่อนที่จะส่งให้ผู้ดูแลระบบ

ผู้ไม่หวังดีดำเนินการทดสอบเข้าสู่ระบบด้วยรหัสผ่าน code4sec

จะเห็นได้ว่าไม่สามารถเข้าใช้งานได้ ผู้ไม่หวังดีดำเนินส่ง URL ให้ผู้ดูแลระบบเข้าใช้งาน

เมื่อผู้ดูแลระบบเข้าใช้งานเว็บ ทดสอบเข้าระบบโดยใช้รหัสผ่านเป็น code4sec
